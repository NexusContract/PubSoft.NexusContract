# ADR-011 URL 资源寻址与复合 ID

**状态**: 提议
**日期**: 2026-01-11
**决策者**: 架构组
**相关**: `ADR-010.md`

---

## 1. 上下文 (Context)
- **现状**: 多维标识在查询和运行时解析复杂且耗时。
- **痛点**: 动态查找导致运行期开销与复杂度。

## 2. 决策 (Decision)
- **核心逻辑**: 在 URL 路径中编码复合 ID（如 `spid-appid`），以支持 O(1) 内存寻址与物理隔离，注意编码长度与边界。

## 3. 理由 (Rationale)
- **业务收益**: 简化寻址、提高查找效率。
- **技术权衡**: URL 长度与编码策略需兼顾安全和可读性。

## 4. 影响与约束 (Impact)
- **副作用**: 需要统一编码规范与解码库。
- **容错底线**: 非法或超长 ID 应返回 4xx，并记录审计信息。
