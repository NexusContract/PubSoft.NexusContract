# ADR-013 Realm 与 Profile 抽象

**状态**: 提议
**日期**: 2026-01-11
**决策者**: 架构组
**相关**: `BLUEPRINT.md`

---

## 1. 上下文 (Context)
- **现状**: 使用渠道特定术语（如 AppId）导致概念混淆。
- **痛点**: 不利于通用化与多商家支持。

## 2. 决策 (Decision)
- **核心逻辑**: 引入 `RealmId`（系统域）与 `ProfileId`（应用/商家），作为通用抽象层以替代渠道特定术语。

## 3. 理由 (Rationale)
- **业务收益**: 去耦合渠道概念，便于多租户扩展。
- **技术权衡**: 需在现有代码中做名词替换与映射。

## 4. 影响与约束 (Impact)
- **副作用**: 文档与接口需更新以反映新命名。
- **容错底线**: 保持向后兼容的映射策略以支持老接口。
